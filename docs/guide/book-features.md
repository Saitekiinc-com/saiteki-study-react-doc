# 📚 書籍探索・読書感想文機能 利用ガイド

このガイドでは、GitHub Issuesを使った「書籍探索」と「読書感想文」機能の使い方を説明します。
GitHubに慣れていない方でも、簡単なフォーム入力だけで利用できます。

## 🔄 なぜこの機能を使うのか？（重要）

私たちのチームには、**「読書感想文が集まるほど、書籍探索が賢くなる」** という仕組み（エコシステム）があります。

1.  あなたが **「読書感想文」** を書いてナレッジベースに登録します。
2.  **「書籍探索」** AIがその感想文（良かった点・難しかった点）を学習します。
3.  次回、他のメンバーが本を探すとき、AIは過去の感想文を参考にして、**より私たちのチームに合った、精度の高い本** を提案できるようになります。

あなたの感想文ひとつが、チーム全体の学習効率を高める貴重なデータになります。ぜひ積極的に活用してください！

---

## 🔍 書籍探索 (Book Search)

「新しい技術を学びたいけど、どの本が良いかわからない」「今の課題解決に役立つ本を知りたい」という時に使います。AIがあなたの状況に合わせて、最適な書籍を3冊提案してくれます。

### 手順

1.  リポジトリ上部の **[Issues]** タブをクリックします。
2.  右上の緑色のボタン **[New Issue]** をクリックします。
3.  リストの中から **[📚 書籍探索依頼]** を見つけ、クリックします。
4.  入力フォームが表示されるので、以下の項目を埋めます。
    *   **お名前**: ニックネームでもOKです。
    *   **役割**: 現在の職種（例: Frontend Engineer）。
    *   **経験年数**: おおよそで構いません。
    *   **達成したい目標**: ここを具体的に書くと、より良い提案が得られます。（例: 「Reactの基礎はわかるので、パフォーマンスチューニングの専門書が読みたい」など）
    *   **わかっていること / わかっていないこと**: 現状のスキルレベルをAIに伝えます。
5.  最後に **[Submit New Issue]** をクリックします。

### 結果

数分以内に、AIがそのIssueにコメントする形で、おすすめの書籍リストを提案してくれます。
提案にはAmazonリンクや、社内のナレッジベースとい照らし合わせた推奨理由が記載されています。

## 📝 読書感想文 (Book Report)

本を読んだら、その学びをチームに共有しましょう。これがAIの賢さにも直結します。

### 手順

1.  リポジトリ上部の **[Issues]** タブをクリックします。
2.  右上の緑色のボタン **[New Issue]** をクリックします。
3.  リストの中から **[📚 読書感想文]** を見つけ、クリックします。
4.  入力フォームに従って感想を書きます。
    *   **書籍名**: 読んだ本の正確なタイトルを入力してください。
    *   **著者**: 著者の名前を入力します。
    *   **リンク**: Amazonや公式サイトのURLを貼ってください。
    *   **読む前の目的 (Objective)**: なぜこの本を読もうと思ったのか、何を解決したかったのかを書きます。
    *   **得られた知識・気づき (Key Takeaways)**: この本を読んで何がわかったか、目的は達成されたかを書きます。
    *   **👍 Positive (良かった点・学び)**: どんなスキルレベルの人におすすめか、実務でどう役立ったかなど。
    *   **👎 Negative (難しかった点・合わなかった点)**: わかりにくかった章や、前提知識が必要だった点など。（正直な感想がAIの精度向上に役立ちます！）
    *   **💡 どんな人におすすめ？**: 具体的にどんな悩みを持つ人に役立つかを書きます。
5.  最後に **[Submit New Issue]** をクリックします。

### 結果

1.  **Pull Requestの作成**: Issueを送信すると、自動的に「Pull Request」が作成されます。
2.  **上司によるレビュー**: 作成されたPull Requestを上司（管理者）が確認し、レビューを行います。
3.  **ナレッジ化**: 上司が承認（マージ）すると、あなたの感想文が正式にナレッジベースとして保存され、チーム全体で共有されます。
