# Lv.2 応用: アプリケーション開発と動作保証

Lv.2のゴールは、<strong>「フロントエンドとバックエンドを連携させ、要件を満たすアプリケーションを独力で実装できること」</strong>です。
UIの構築からAPIの実装、データベースの操作までを一気通貫で行い、エラーが発生しても自力で解決できる能力を養います。

## 1. 実践的なUI実装 (Frontend Focus)
ユーザーが直感的に操作できるインターフェースを構築します。

### 学習項目
*   **フォームハンドリング**:
    *   React Hook Form などのライブラリを使ったバリデーション（必須、文字数、形式）。
    *   制御コンポーネント (Controlled) と非制御コンポーネント (Uncontrolled) の理解。
*   **ルーティング**:
    *   動的ルーティング (`/users/:id`) とネストされたルート。
    *   保護されたルート（ログイン必須ページ）の実装。
*   **API連携UI**:
    *   ローディング状態 (`isLoading`) とエラー状態 (`isError`) の適切な表示。
    *   `try-catch` によるエラーハンドリングとユーザーへのフィードバック。

### チェックリスト
- [ ] フォームのバリデーションを実装し、エラー時に適切なメッセージを表示できる。
- [ ] URLパラメータを取得して、それに応じたデータを表示できる。
- [ ] APIリクエスト失敗時に、ユーザーに「何が起きたか」を伝えるUIを作れる。

## 2. APIとデータベースの実装 (Backend Focus)
フロントエンドからのリクエストを受け付け、データを永続化します。

### 学習項目
*   **API実装**:
    *   RESTful APIの構築（URL設計、メソッドの使い分け）。
    *   リクエストパラメータ (`query`, `params`, `body`) の取得とバリデーション。
    *   **重要**: フロントエンドのバリデーションは「親切」のため、バックエンドのバリデーションは「安全」のため。必ず両方で行う。
*   **データベース操作 (CRUD)**:
    *   ORM (Prisma/TypeORMなど) を使ったデータベース操作。
    *   マイグレーション管理（スキーマ変更の履歴管理）。
    *   リレーション（1対多など）を考慮したデータの取得・保存。

### チェックリスト
- [ ] ユーザー登録APIで、メールアドレスの重複チェックや形式チェックを実装できる。
- [ ] マイグレーションファイルを作成し、DBにテーブルを作成できる。
- [ ] 関連するテーブル（例：ユーザーと投稿）を結合してデータを取得できる。

## 3. デバッグとトラブルシューティング (Fullstack)
「動かない」ときに、システム全体を見て原因を特定します。

### 学習項目
*   **フロントエンドのデバッグ**:
    *   React Developer ToolsでのState/Props確認。
    *   Networkタブでのリクエスト/レスポンス確認。
*   **バックエンドのデバッグ**:
    *   構造化ロギング（リクエストID、タイムスタンプ、スタックトレース）。
    *   例外処理 (`try-catch`) と適切なHTTPステータスコードの返却。

### チェックリスト
- [ ] エラー発生時、ブラウザのNetworkタブとサーバーのログを見比べて、問題がどこ（Front/Back/DB）にあるか特定できる。
- [ ] 存在しないリソースへのアクセスに対して、適切に 404 エラーを返せる。

## 推奨学習リソース
*   **[React Hook Form](https://react-hook-form.com/jp/)**: フォームバリデーション。
*   **[React Router](https://reactrouter.com/en/main/start/tutorial)**: ルーティング。
*   **[Prisma](https://www.prisma.io/docs)**: モダンなORM。
