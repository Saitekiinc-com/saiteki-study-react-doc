# Lv.2 応用: アプリ開発・動作保証

Lv.2のゴールは、**「エラーなくアプリを開発し、動作することを確認できること」**です。
仕様書や要件定義に基づき、バグのない機能を実装し、問題が発生しても自力で解決できる能力を養います。

## 1. 実践的なコンポーネント実装
基礎で学んだことを組み合わせ、複雑なUIを構築します。

### 学習項目
*   **フォームハンドリング**:
    *   React Hook Form などのライブラリを使ったバリデーション。
    *   制御コンポーネント (Controlled) と非制御コンポーネント (Uncontrolled) の違い。
*   **ルーティング (React Router)**:
    *   動的ルーティング (`/users/:id`)。
    *   ネストされたルート (`Outlet`)。
    *   保護されたルート（ログイン必須ページなど）。
*   **API連携の実践**:
    *   ローディング状態 (`isLoading`) とエラー状態 (`isError`) の適切なUI表示。
    *   `try-catch` によるエラーハンドリング。

### チェックリスト
- [ ] フォームのバリデーション（必須、文字数、形式チェック）を実装できる。
- [ ] URLパラメータを取得して、それに応じたデータを表示できる。
- [ ] APIリクエストが失敗した際に、ユーザーに適切なエラーメッセージを表示できる。

## 2. デバッグとトラブルシューティング
「動かない」ときに何を見るべきかを知っています。

### 学習項目
*   **React Developer Tools**:
    *   コンポーネントツリーの確認。
    *   PropsとStateの現在の値の確認。
    *   「なぜ再レンダリングされたか」の追跡 (Profiler)。
*   **ブラウザのデバッガ**:
    *   `debugger` ステートメントやブレークポイントの使用。
    *   NetworkタブでのAPIリクエスト/レスポンスの確認。

### チェックリスト
- [ ] React DevToolsを使って、特定のコンポーネントのStateを書き換えられる。
- [ ] Networkタブを見て、APIリクエストのヘッダーやボディが正しいか確認できる。
- [ ] エラーログ（スタックトレース）を見て、問題の発生箇所を特定できる。

## 3. 要件を満たす実装
「なんとなく動く」ではなく「仕様通りに動く」を目指します。

### 学習項目
*   **エッジケースの考慮**:
    *   データが空の場合 (Empty State)。
    *   データが大量にある場合 (Pagination/Infinite Scroll)。
    *   ネットワークが遅い場合。
*   **ライブラリの選定と利用**:
    *   UIライブラリ (MUI, Chakra UI, Tailwind UI) の活用。
    *   日付操作 (date-fns, dayjs) などのユーティリティ。

### チェックリスト
- [ ] データが0件のときに「データがありません」と表示するなどの配慮ができている。
- [ ] 既存のライブラリを使って、カレンダーやモーダルなどの複雑なUIを実装できる。
