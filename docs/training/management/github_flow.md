# GitHubを活用した研修管理ガイド

この研修では、実務での開発フローに慣れるため、GitHubの機能（Issues, Milestones, Projects）を使って学習進捗を管理します。

## 1. マイルストーン (Milestones) の設定
メンター（または管理者）は、学習の区切りごとにマイルストーンを作成してください。

*   **作成単位の例**:
    *   レベル別: `Lv.1 基礎`, `Lv.2 応用`...
    *   期間別: `Month 1`, `Month 2`...
*   **期限 (Due date)**: 目標とする完了日を設定します。

## 2. イシュー (Issues) の活用
研修生は、学習を開始する際にIssueを作成します。用意されているテンプレートを使用してください。

### 📚 学習タスク (Learning Task)
カリキュラムのモジュールごとに作成します。
1.  **Title**: `[Lv.1] Web標準の仕組み` のように入力。
2.  **Milestone**: 対応するマイルストーンを設定。
3.  **Assignees**: 自分自身を割り当て。
4.  **Checklist**: カリキュラムページのチェックリストをコピペして、進捗を管理。

### ❓ 質問・相談 (Question)
学習中に詰まったら作成します。
*   「何が起きていて」「何を試したか」を言語化する練習になります。
*   解決したら、解決方法をコメントに残してCloseします（ナレッジベースになります）。

### 📅 週次振り返り (Weekly Report)
毎週の終わりに作成します。
*   KPT (Keep, Problem, Try) フレームワークを使って振り返りを行います。
*   メンターはこれを見てフィードバックを行います。

## 3. プロジェクト (Projects) の活用
Issueをカンバン形式（ボード）で表示するには、GitHub Projects機能を使用します。

### カンバンボードの作成手順
1.  リポジトリ上部の **Projects** タブをクリックします。
2.  **Link a project** または **New project** をクリックします（初めての場合は「New project」）。
3.  **New project** を選択し、テンプレートから **Board** を選択して **Create** をクリックします。
4.  プロジェクト名（例: `Training Progress`）を入力します。

### カンバンボードの設定
作成したボードで、以下の設定を行うと便利です。

1.  **ステータスの定義**:
    *   デフォルトで `Todo`, `In Progress`, `Done` がありますが、必要に応じて `Review` (レビュー待ち) などを追加します。
2.  **Issueの追加**:
    *   ボード下部の「+ Add item」から、既存のIssueを検索して追加できます（`#` を入力すると候補が出ます）。
    *   または、各Issueの右サイドバーにある **Projects** 項目から、このプロジェクトを選択すると自動的に追加されます。
3.  **自動化 (Workflows)**:
    *   プロジェクト設定の「Workflows」を使うと、「IssueがCloseされたら自動でDoneに移動する」といった自動化が可能です。
